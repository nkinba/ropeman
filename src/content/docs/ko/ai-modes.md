Ropeman은 상황에 맞게 고를 수 있는 **4가지 AI 트랙**을 제공합니다. 각 모드는 비용, 성능, 보안, 설정 난이도가 다르므로, 목적에 맞는 트랙을 선택하는 것이 중요합니다.

## Demo

가장 빠르게 체험할 수 있는 무료 모드입니다.

- **특징** — 회원가입이나 API 키 없이 즉시 사용 가능합니다. 요청 횟수와 프로젝트 크기에 완만한 제한이 있습니다.
- **요구사항** — 없음. 브라우저만 있으면 됩니다.
- **데이터 흐름** — 브라우저 → Ropeman 엣지 프록시 → AI 제공사. 원본 소스 코드는 전송되지 않고, **코드 구조 요약(기본 250 KB 한도)** 만 전달됩니다.
- **추천 상황** — 처음 Ropeman을 써 보는 경우, 빠르게 데모를 보여줘야 하는 경우, 중소형 프로젝트 체험.

## API Key (BYOK)

본인의 LLM API 키를 쓰는 Bring Your Own Key 방식입니다.

- **특징** — Google Gemini, Anthropic Claude, OpenAI GPT 중 원하는 공급자를 선택해 더 강력한 모델을 사용할 수 있습니다. 속도 제한은 해당 공급자 계정 기준으로 적용됩니다.
- **요구사항** — 하나 이상의 LLM API 키. 키는 **브라우저 localStorage** 에만 저장되며 Ropeman 서버로 보내지지 않습니다.
- **데이터 흐름** — 브라우저 → Ropeman 엣지 프록시 → AI 제공사. 프록시는 키를 저장하지 않고 단순 중계만 수행합니다. 전송되는 페이로드는 여전히 코드 구조 요약입니다.
- **추천 상황** — 더 큰 프로젝트, 더 정확한 분석 결과가 필요한 경우, 이미 LLM API 키를 보유한 개발자.

## Bridge (로컬 CLI 연동)

로컬에 설치된 AI CLI를 Ropeman과 연결하는 방식입니다.

- **특징** — Claude Code, Gemini CLI 등 로컬 CLI의 인증을 그대로 활용합니다. 별도 API 키 입력이 필요 없고, **요청이 Ropeman 서버를 거치지 않고** 사용자의 CLI에서 직접 AI 제공사로 갑니다.
- **요구사항** — Bridge 서버 실행 + 지원되는 로컬 CLI 설치. 자세한 설치는 [Bridge 서버 설정](/docs/ko/bridge-setup)을 참고하세요.
- **데이터 흐름** — 브라우저 → 로컬 Bridge 서버(localhost) → 로컬 CLI → AI 제공사. 중간에 Ropeman 인프라가 전혀 관여하지 않습니다.
- **추천 상황** — 회사 정책상 외부 프록시를 쓸 수 없는 환경, 이미 Claude Code 혹은 Gemini CLI를 구독 중인 사용자, 최대 수준의 데이터 통제가 필요한 경우.

## WebGPU (브라우저 내장 AI, 실험적)

브라우저 안에서 LLM을 직접 실행합니다.

- **특징** — 브라우저 내장 LLM 런타임을 사용해 **완전 오프라인** 으로 동작합니다. 외부 네트워크 요청이 발생하지 않으며, 한 번 다운로드한 모델은 브라우저에 캐시돼 재사용됩니다.
- **요구사항** — WebGPU를 지원하는 최신 브라우저와 충분한 VRAM/RAM. 최초 실행 시 모델 파일(수백 MB)을 다운로드합니다.
- **데이터 흐름** — 모든 추론이 사용자 기기 안에서 끝납니다. 어떤 데이터도 외부로 나가지 않습니다.
- **추천 상황** — 완전 오프라인 환경, 보안이 최우선인 경우, 성능 테스트가 가능한 고사양 기기. 소형 프로젝트부터 시도해 보시기 바랍니다.

## 코드 구조 요약 크기와 분석 품질

모든 AI 트랙에서 Ropeman은 원본 소스 대신 **코드 구조 요약(skeleton)** 만 모델에 전송합니다. 이 요약의 크기는 분석 품질과 속도, 그리고 사용 가능한 모델에 직접 영향을 줍니다.

- **클수록 정확하고, 작을수록 빠릅니다.** 요약이 크면 모델이 더 많은 함수·클래스·관계를 한 번에 볼 수 있어 다이어그램이 더 정교해지지만, 토큰 수가 늘어나는 만큼 응답 시간과 비용도 함께 증가합니다.
- **모델마다 한도가 다릅니다.** 같은 트랙이라도 선택한 모델의 컨텍스트 윈도우와 사양에 따라 처리 가능한 요약 크기와 응답 속도가 달라집니다. 특히 WebGPU 모드의 소형 모델은 다른 트랙보다 훨씬 작은 입력만 처리할 수 있습니다.
- **기본값과 조정.** 기본 한도는 **250 KB** 이며, **Settings → 코드 구조 요약 크기 제한** 에서 줄이거나 늘릴 수 있습니다. API 호출이 토큰 한도 오류로 실패하면 값을 줄이고, 더 정교한 분석이 필요하면 늘리세요. WebGPU에서는 약 12 KB 정도가 안정적입니다.

대부분의 사용자는 기본값으로 시작한 뒤, 결과 품질이 부족하면 단계적으로 늘려 보는 흐름을 권장합니다.

## 모드 선택 가이드

| 기준               | Demo  | API Key    | Bridge   | WebGPU           |
| ------------------ | ----- | ---------- | -------- | ---------------- |
| 설정 난이도        | 없음  | 낮음       | 중간     | 낮음             |
| 비용               | 무료  | API 사용량 | CLI 구독 | 무료             |
| 성능(모델 품질)    | 중    | 상         | 상       | 기기 사양에 따름 |
| Ropeman 서버 경유  | 예    | 예(프록시) | 아니오   | 아니오           |
| 오프라인 동작      | 불가  | 불가       | 일부     | 완전             |
| 추천 프로젝트 크기 | 소~중 | 소~대      | 소~대    | 소               |

어떤 모드를 선택하든 **원본 소스 코드는 브라우저를 떠나지 않는다는 원칙**은 동일합니다. 자세한 데이터 흐름은 [보안 문서](/docs/ko/security)에서 확인할 수 있습니다.
