# Public Gallery (`/explore`) Refresh Guide

The `/explore` gallery shows AI-generated architecture snapshots of curated
open-source projects (React, Vue, Svelte, FastAPI, Gin, …). Per
[ADR-19](../.spec/ARCH.md), these snapshots are **manually refreshed** rather
than auto-generated by a Cloudflare Cron — refreshing them is a 5–15 minute
job done a few times a year.

## One-time setup

1. Create the KV namespace:

   ```sh
   cd edge-proxy
   wrangler kv namespace create ROPEMAN_EXPLORE_STORE -c wrangler.explore.toml
   ```

   Paste the returned `id` into `wrangler.explore.toml` (replace `REPLACE_WITH_KV_ID`).

2. Set the admin token (used for `POST` / `DELETE`):

   ```sh
   # Generate a random token, then:
   wrangler secret put EXPLORE_ADMIN_TOKEN -c wrangler.explore.toml
   ```

3. Deploy the worker:

   ```sh
   npm run deploy:explore
   # or from repo root: npm run deploy:edge:explore
   ```

4. Verify:
   ```sh
   curl https://explore.ropeman.dev/explore   # → { "slugs": [] }
   ```

## Refreshing a snapshot

Repeat for each entry in `src/lib/data/explore-curated.json`.

1. **Analyze locally**: open https://ropeman.dev (or the local dev server),
   paste the GitHub URL of the curated repo (e.g. `facebook/react`), wait for
   AI analysis to complete, drill into the most informative sub-domains so the
   cache is populated.

2. **Capture the snapshot**: open the browser devtools console and run

   ```js
   __snap();
   ```

   (or build the snapshot manually from `__semanticStore.cache`). The snapshot
   shape:

   ```json
   {
     "projectName": "react",
     "owner": "facebook",
     "repo": "react",
     "analyzedAt": "2026-04-13",
     "filePaths": ["..."],
     "semanticLevels": {
       "__root__": { "parentId": null, "nodes": [...], "edges": [...], "breadcrumbLabel": "react" },
       "sem:reconciler": { ... }
     }
   }
   ```

3. **Upload to KV** via the explore worker:

   ```sh
   curl -X POST https://explore.ropeman.dev/explore/react \
     -H "Authorization: Bearer $EXPLORE_ADMIN_TOKEN" \
     -H "Content-Type: application/json" \
     -d @react-snapshot.json
   ```

   The worker accepts payloads up to 2 MB and validates JSON structure on the
   client side at fetch time (`exploreService.validateSnapshot`).

4. **Verify** at https://ropeman.dev/explore/react.

## Adding a new curated entry

1. Edit `src/lib/data/explore-curated.json` and append a new object:
   ```json
   {
   	"slug": "django",
   	"owner": "django",
   	"repo": "django",
   	"title": "Django",
   	"language": "Python",
   	"description": "The web framework for perfectionists with deadlines.",
   	"stars": "82k+"
   }
   ```
2. Update `updatedAt` to today.
3. Run `npm run build` — the new slug is auto-included in the prerendered
   `/explore/[slug]` route via `entries()` and added to `sitemap.xml`.
4. Refresh the snapshot following the steps above.

## Removing an entry

1. Delete the entry from `src/lib/data/explore-curated.json`.
2. Delete the KV entry:
   ```sh
   curl -X DELETE https://explore.ropeman.dev/explore/<slug> \
     -H "Authorization: Bearer $EXPLORE_ADMIN_TOKEN"
   ```
3. Rebuild & redeploy.

## Cost & limits

- KV reads are free (cached at the edge); each fetch is `Cache-Control: public, max-age=300, s-maxage=3600`.
- KV storage is ~$0.50/GB/month — six 200KB snapshots cost essentially $0.
- The admin token is the only thing standing between malicious uploads and KV;
  rotate it if it leaks: `wrangler secret put EXPLORE_ADMIN_TOKEN -c wrangler.explore.toml`.

## CORS policy note

The explore worker's `ALLOWED_ORIGINS` allowlist applies **only when the
request has an `Origin` header** — i.e. when a browser is calling from
another site. Server-to-server callers (curl, CI, admin scripts) do not
send `Origin`, so the allowlist is skipped for them and the request
proceeds straight to routing. For admin `POST` / `DELETE`, the Bearer
token is the real authorization check; the `Origin` allowlist only
guards browser traffic that would otherwise bypass CORS. This means
the `curl` examples above work without needing `-H "Origin: ..."`.
